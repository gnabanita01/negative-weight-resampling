# negative-weight-resampling
project overview :
This repository contains solution for GSOC pre selection task focused on "Negative weight mitigation".

Goal : Eliminate Negative weights generated by NLO calculations using Cell Resampling method to reduce statistical dilution in downstream simulation.
Implementation Details : 
1. Born Projection: Explains how to map (n+1)-body real emission events into the n-body phase space using the formula pT = p{T,real} + z{gluon}.
2. Algorithm Logic: a) dynamic cell building process, b) finding nearest neighbour in (pT, y) space using a distance matric, c) redistributing weight once the cell sum weight is non-negative.
3. Computation Complexity : Nested loop is used here so time complexity is O(N.N)

Discussion on Scaling Factor : pT and y have different numerical ranges such as for pT(momentum) it would range from tens to thousands of units but for y(rapidity) it would be from -4 to +4, so without the factor 100, the pT difference would dominate the distance calculation making the rapidity distribution negligible and lose physical accuracy. If standard Euclidean distance is used, a difference of 1 unit in pT would be treated the same as 1 unit in y. However, 1 unit in y is a massive physical shift, while 1 unit in pT is negligible.

Infinite Event Limit: as it is approaching to infinite number of events, the phase space becomes so dense the distance to nearest neighbour shrinks towards zero, the cells become smaller. Here, the need for scaling factor decreases because there will be always a neighbour who is extremely close in both pT and y.
